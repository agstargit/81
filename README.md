# 聚餐管理 · 可在线编辑版（云端）

把原来"存在浏览器本机"的聚餐管理页面，升级为**云端在线编辑**：任何设备打开同一个链接都能编辑，修改自动保存到服务器 `data.json`，换手机/电脑打开还是同一份数据。

## 运行（任选其一）

### 1）本地 / 自有服务器（Node ≥ 16，零依赖）
```bash
cd deploy
node server.js            # 默认端口 3000，可用 PORT 环境变量改：PORT=8080 node server.js
# 浏览器打开 http://localhost:3000
```
- 首次启动会自动生成 `data.json`（初始示例数据）。
- 编辑页面后点击任意修改会自动 POST 到 `/data` 落盘；顶部会显示「☁ 云端已同步」。
- 云端不可用时自动回退到本机浏览器缓存（顶部显示「💾 本地保存」），不会丢数据。

### 2）部署到可运行 Node 的云平台（获得公网链接）
把整个 `deploy/` 目录上传到支持 Node 的服务（如 CloudBase 云函数/云托管、Railway、Render、任意 VPS 等），启动命令设为 `node server.js`，开放端口（默认 3000）。部署成功后把公网地址发群里即可。

> 说明：原 CloudStudio 静态托管只能读、不能写，因此本版需要一个能跑 Node 进程的环境。若继续用静态托管，页面仍可打开，但数据会回退为"仅本机保存"（每台设备各存各的）。

## 目录结构
```
deploy/
├─ server.js        # 零依赖 Node 后端：托管页面 + 读写 /data
├─ public/
│  └─ index.html    # 前端单页（已改为云端优先读写）
├─ data.json        # 云端数据（首次自动生成，日常自动更新）
├─ package.json
└─ README.md
```

## 数据接口
- `GET  /data` → 返回当前数据 JSON（页面加载时拉取，作为权威数据）
- `POST /data` → 保存数据（前端每次修改后自动调用，带写锁防并发覆盖）

## 恢复 / 备份
- 想重置为初始示例：停服后删除 `data.json`，重启即重新生成种子数据（页面内「恢复初始示例」按钮同样可重置）。
- 想备份：直接复制 `data.json`，或用页面「导出数据」下载 JSON。
